# Cost Optimization for Agents

## Overview

LLM API calls cost money. At scale, an unoptimized agent can burn through budgets quickly. This chapter covers five proven strategies to reduce your LLM costs by 40-70% without sacrificing quality: prompt compression, semantic caching, model routing, response streaming, and cost tracking. Every strategy is directly applicable to the Course Companion FTE architecture.

## Key Concepts

**Where Do Costs Come From?**
Claude charges per token: input tokens (your prompt + history) + output tokens (Claude's response). The biggest cost drivers are:
1. **Long system prompts** repeated on every call
2. **Full conversation history** sent each turn
3. **Using large models** for simple tasks
4. **Duplicate questions** asked by many users

**Five Optimization Strategies**

| Strategy | Cost Reduction | Complexity |
|----------|---------------|------------|
| Prompt compression | 20-30% | Low |
| Semantic caching | 30-50% | Medium |
| Model routing | 20-40% | Medium |
| History pruning | 10-20% | Low |
| Batch processing | 15-25% | High |

## Deep Dive

**Strategy 1: Prompt Compression**
Shorter prompts = fewer input tokens. Compress by removing filler words and formatting instructions into dense bullet points.
```python
# Before: 150 tokens
system_verbose = """
You are a helpful and patient educational tutor specializing in AI Agent Development.
Your goal is to help students understand complex concepts by using simple analogies.
Please always be encouraging and supportive in your responses.
"""

# After: 40 tokens — same behavior, 73% fewer tokens
system_compressed = """Tutor: AI Agent Dev. Use analogies. Be concise. Encourage students."""
```

**Strategy 2: Semantic Caching**
Cache responses for semantically similar questions. If 100 students ask "What is MCP?", answer once and reuse.
```python
from sentence_transformers import SentenceTransformer
import asyncpg

model = SentenceTransformer("all-MiniLM-L6-v2")
SIMILARITY_THRESHOLD = 0.92  # 92% similar = cache hit

async def get_cached_response(db, question: str) -> str | None:
    """Return cached response if a similar question was answered before."""
    embedding = model.encode(question).tolist()
    row = await db.fetchrow(
        """
        SELECT response, 1 - (embedding <=> $1::vector) AS similarity
        FROM response_cache
        WHERE 1 - (embedding <=> $1::vector) > $2
        ORDER BY embedding <=> $1::vector
        LIMIT 1
        """,
        embedding, SIMILARITY_THRESHOLD
    )
    return row["response"] if row else None

async def cache_response(db, question: str, response: str):
    """Cache a question-response pair."""
    embedding = model.encode(question).tolist()
    await db.execute(
        "INSERT INTO response_cache (question, response, embedding) VALUES ($1, $2, $3::vector)",
        question, response, embedding
    )
```

**Strategy 3: Model Routing**
Use cheaper, faster models for simple tasks and save the expensive models for complex reasoning.
```python
def select_model(task_type: str) -> str:
    """Route to the appropriate model based on task complexity."""
    routing = {
        "simple_qa": "claude-haiku-4-5-20251001",      # ~10x cheaper
        "explanation": "claude-sonnet-4-20250514",  # balanced
        "complex_analysis": "claude-opus-4-6",      # most capable
    }
    return routing.get(task_type, "claude-sonnet-4-20250514")

def classify_task(question: str) -> str:
    """Simple heuristic to classify task complexity."""
    if len(question) < 50 and "?" in question:
        return "simple_qa"
    if any(w in question.lower() for w in ["analyze", "compare", "evaluate", "design"]):
        return "complex_analysis"
    return "explanation"
```

**Strategy 4: History Pruning**
Don't send the full conversation history on every call. Prune to the last N exchanges.
```python
def prune_history(messages: list, max_pairs: int = 5) -> list:
    """Keep only the last N user-assistant pairs."""
    # Always keep the first message (initial context)
    if len(messages) <= max_pairs * 2 + 1:
        return messages
    return messages[:1] + messages[-(max_pairs * 2):]
```

## Code Example

```python
# Cost tracking — log every LLM call
import logging
from datetime import datetime

logger = logging.getLogger(__name__)

async def tracked_api_call(db, user_id: str, feature: str, model: str,
                           messages: list, system: str = "") -> str:
    """Make an API call and log cost to database."""
    import anthropic
    client = anthropic.Anthropic()

    response = client.messages.create(
        model=model,
        max_tokens=1024,
        system=system,
        messages=messages,
    )

    # Calculate cost (approximate — check Anthropic pricing page for current rates)
    COST_PER_1K = {"claude-haiku-4-5-20251001": 0.00025, "claude-sonnet-4-20250514": 0.003, "claude-opus-4-6": 0.015}
    rate = COST_PER_1K.get(model, 0.003)
    total_tokens = response.usage.input_tokens + response.usage.output_tokens
    cost_usd = (total_tokens / 1000) * rate

    # Log to database
    await db.execute(
        """INSERT INTO cost_logs (user_id, feature, model, input_tokens, output_tokens, cost_usd, created_at)
           VALUES ($1, $2, $3, $4, $5, $6, $7)""",
        user_id, feature, model,
        response.usage.input_tokens, response.usage.output_tokens,
        cost_usd, datetime.utcnow()
    )

    logger.info("LLM call: user=%s feature=%s tokens=%d cost=$%.4f",
                user_id, feature, total_tokens, cost_usd)

    return response.content[0].text
```

## Summary

- Prompt compression cuts input tokens 20-30% with no quality loss
- Semantic caching eliminates duplicate API calls for similar questions
- Model routing: use Haiku for simple QA, Sonnet for explanations, Opus for analysis
- History pruning: keep last 5 exchanges, not the entire conversation
- Always log tokens and cost per call — budget surprises are avoidable
- Combine all five strategies for maximum 60-70% cost reduction
- Cost logging is required in Phase 2 before any LLM call (constitutional rule)
