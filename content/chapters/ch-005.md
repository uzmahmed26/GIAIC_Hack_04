# Tool Use and Function Calling

## Overview

Tool use is the mechanism that transforms a passive LLM into an active agent. When Claude receives a message, it can decide to call a function (a "tool") instead of — or before — generating a text response. This chapter explains how tool use works, how to define tools, and how to build the agentic loop that handles tool calls end to end.

## Key Concepts

**What Is a Tool?**
A tool is a function definition you pass to Claude. Claude reads the name, description, and input schema — then decides whether to call it. Your code executes the actual function; Claude only decides *when* to call it and *what arguments* to pass.

**The Agentic Loop**
The standard tool use loop:
1. Send message to Claude with tool definitions
2. Claude responds with `tool_use` (or `text` if no tool needed)
3. Your code executes the tool and captures the result
4. Send `tool_result` back to Claude as a user message
5. Claude generates a final text response using the result
6. Repeat if Claude calls another tool

**Stop Reasons**
- `end_turn` — Claude is done, no more tools needed
- `tool_use` — Claude wants to call a tool; your loop must continue

## Deep Dive

**Defining a Tool**
```python
search_tool = {
    "name": "search_course_content",
    "description": "Search course chapters for content relevant to the student's question. Always call this before explaining a concept.",
    "input_schema": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "The search query based on the student's question"
            },
            "chapter_id": {
                "type": "string",
                "description": "Optional: limit search to a specific chapter (e.g., 'ch-003')"
            },
            "limit": {
                "type": "integer",
                "description": "Number of results to return (1-10, default 3)",
                "default": 3
            }
        },
        "required": ["query"]
    }
}
```

**The Complete Agentic Loop**
```python
import anthropic
import httpx

client = anthropic.Anthropic()
API_BASE = "http://localhost:8000/api/v1"

def call_tool(tool_name: str, tool_input: dict) -> str:
    """Execute a tool call and return the result as a string."""
    if tool_name == "search_course_content":
        params = {"q": tool_input["query"], "limit": tool_input.get("limit", 3)}
        if "chapter_id" in tool_input:
            params["chapter"] = tool_input["chapter_id"]
        response = httpx.get(f"{API_BASE}/search", params=params)
        return response.text
    raise ValueError(f"Unknown tool: {tool_name}")

def run_agent(user_message: str, tools: list) -> str:
    """Run the agentic loop until Claude produces a final text response."""
    messages = [{"role": "user", "content": user_message}]

    while True:
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            tools=tools,
            messages=messages,
        )

        if response.stop_reason == "end_turn":
            # Extract final text response
            return next(b.text for b in response.content if b.type == "text")

        if response.stop_reason == "tool_use":
            # Add Claude's response to history
            messages.append({"role": "assistant", "content": response.content})

            # Execute all tool calls in this response
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    result = call_tool(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result,
                    })

            # Add tool results and continue loop
            messages.append({"role": "user", "content": tool_results})
```

## Code Example

```python
# Full example: agent that uses search before explaining
tools = [search_tool]  # defined above

answer = run_agent(
    "What is the difference between an MCP server and an MCP client?",
    tools
)
print(answer)
# Claude will call search_course_content first, then explain using the results
```

## Summary

- Tool use lets Claude call functions instead of guessing answers
- Every tool needs: name, description, input_schema (JSON Schema)
- The agentic loop runs until `stop_reason == "end_turn"`
- Tool results are sent back as `role: user` messages
- Multiple tools can be called in a single response
- Your code executes tools; Claude only decides when and with what arguments
