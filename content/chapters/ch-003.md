# MCP Protocol Deep Dive

## Overview

MCP (Model Context Protocol) is an open standard that defines how AI agents connect to external tools and data sources. Instead of writing custom integration code for every service, MCP gives you a universal plug-and-play interface — your agent connects to an MCP server, and the server exposes tools the agent can call. This chapter explains how MCP works, why it exists, and how to connect it to Claude.

## Key Concepts

**Why MCP Exists**
Before MCP, every AI team had to write custom code to connect LLMs to their databases, file systems, and APIs. This created fragmentation — every integration was different, hard to test, and impossible to reuse. MCP solves this by defining a single protocol that any AI model and any tool can implement.

**MCP Servers and Clients**
- **MCP Server** — a process that exposes a set of tools (e.g., "read a file", "run a query"). Servers are standalone programs that run alongside your agent.
- **MCP Client** — the component inside your agent that connects to MCP servers and calls their tools. Claude SDK acts as the MCP client.

**Three MCP Servers Used in This Project**
| Server | Purpose | Phase |
|--------|---------|-------|
| `filesystem` | Read/write local files | Phase 2+ |
| `fetch` | HTTP requests and R2 URL retrieval | Phase 2+ |
| `postgres` | Direct PostgreSQL queries | Phase 3 only |

## Deep Dive

**Starting an MCP Server**
MCP servers ship as npm packages and run with npx:
```bash
# Start filesystem MCP server (gives agent access to ./content and ./SKILLS)
npx -y @modelcontextprotocol/server-filesystem ./content ./SKILLS
```

**Connecting Claude to an MCP Server**
```python
import anthropic

client = anthropic.Anthropic()

# Define an MCP tool manually (SDK handles this automatically with MCP)
tools = [
    {
        "name": "read_file",
        "description": "Read the content of a local file",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "File path to read"}
            },
            "required": ["path"]
        }
    }
]

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[{"role": "user", "content": "Read the file content/chapters/ch-001.md"}],
)
```

**The MCP Config File**
In this project, all MCP server definitions live in `mcp-config/servers.json`. Start all Phase 2 servers using:
```powershell
# Windows
powershell -ExecutionPolicy Bypass -File mcp-config/start-mcp.ps1
```

## Code Example

```python
# Practical MCP usage with Claude Agent SDK
# This shows the pattern — actual MCP client integration
# uses the claude_agent_sdk package

import anthropic

client = anthropic.Anthropic()

# Tool call result (what MCP server returns after agent calls a tool)
tool_result = {
    "role": "user",
    "content": [
        {
            "type": "tool_result",
            "tool_use_id": "toolu_01234",
            "content": "# Chapter 1\nIntroduction to AI Agents..."
        }
    ]
}

# Agent processes the tool result and responds
messages = [
    {"role": "user", "content": "Read chapter 1 and summarize it"},
    {"role": "assistant", "content": [{"type": "tool_use", "id": "toolu_01234", "name": "read_file", "input": {"path": "ch-001.md"}}]},
    tool_result,
]

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=512,
    messages=messages,
)
print(response.content[0].text)
```

## Summary

- MCP = standard protocol for connecting AI agents to external tools
- MCP servers expose tools; MCP clients (Claude SDK) call those tools
- Three project MCP servers: `filesystem`, `fetch`, `postgres`
- MCP servers run as separate processes started with npx
- MCP config lives in `mcp-config/servers.json`
- MCP is used in Phase 2 and Phase 3 only — not in Phase 1
