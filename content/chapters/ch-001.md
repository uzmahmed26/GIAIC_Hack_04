# Introduction to AI Agents

## Overview

An AI agent is a software system that can perceive its environment, reason about it, and take actions to achieve a goal — autonomously and repeatedly. Unlike a simple LLM that answers one question at a time, an agent can plan multi-step tasks, use tools, remember context, and course-correct when things go wrong. In 2026, AI agents are transforming every industry by operating as Digital Full-Time Equivalents (FTEs) that work 168 hours per week.

## Key Concepts

**LLM vs AI Agent**
An LLM (Large Language Model) is a text-in, text-out system. You give it a prompt, it returns a completion. It has no memory across calls, no ability to take actions, and no loop. An AI agent wraps an LLM with a perception layer, a reasoning loop, and an action layer — making it capable of doing real work in the world.

**Three Core Components of an Agent**
1. **Perception** — the agent observes its environment (user messages, tool results, database reads, web searches)
2. **Reasoning** — the LLM inside the agent decides what to do next based on what it perceives
3. **Action** — the agent executes something: calls a tool, writes to a file, sends a message, queries a database

**Agent Loop**
Agents run in a loop: perceive → reason → act → perceive again. This loop continues until the goal is reached or a stopping condition is met. Each iteration can produce new information that changes the agent's next decision.

## Deep Dive

**Why Agents Matter in 2026**
The shift from LLMs to agents is the shift from "ask and answer" to "assign and complete." A solo developer using Claude as an agent can now do the work of a small team. Agents can:
- Write code, run tests, fix bugs, and commit — all in one session
- Research a topic, draft a report, and email it to stakeholders
- Monitor a system, detect anomalies, and page on-call — without human intervention

**Digital FTE Concept**
A Digital FTE is an AI agent that replaces the work output of a full-time human employee in a specific domain. Unlike a chatbot that responds to one query, a Digital FTE:
- Maintains persistent goals across sessions
- Manages its own task queue
- Reports progress and blockers
- Escalates to humans only when truly necessary

**Agent Factory Concept**
Agent Factory is an architecture where a General Agent (built on Claude) reads a specification document and manufactures a Custom Agent tailored to a specific job. This course teaches you the full Agent Factory stack — from writing your first agent to deploying a production-grade factory.

## Code Example

```python
import anthropic

client = anthropic.Anthropic()

# A minimal agent loop
messages = []
user_goal = "Research the top 3 MCP servers and summarize them"

messages.append({"role": "user", "content": user_goal})

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system="You are a research agent. Complete the user's goal step by step.",
    messages=messages,
)

print(response.content[0].text)
```

## Summary

- An AI agent = LLM + perception + reasoning + action loop
- The three agent components are: Perception, Reasoning, Action
- Agents run in a loop until the goal is achieved
- A Digital FTE works 168 hours/week on a specific domain
- Agent Factory is an architecture where General Agents create Custom Agents from specs
