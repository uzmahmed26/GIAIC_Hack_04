# Agent Skills Design

## Overview

Agent Skills are markdown files that define how your AI agent behaves in specific situations. Instead of hardcoding behavior in prompts, skills let you modularize your agent's capabilities — each skill is a focused instruction set that gets loaded at runtime. This chapter teaches you how to design, write, and load skills into a ChatGPT App.

## Key Concepts

**What Is a Skill?**
A skill is a structured markdown file that defines:
- A specific role or behavior for the agent in one context
- Which APIs to call and when
- How to format responses
- What guardrails to apply (never do X, always do Y)

**Why Skills Instead of One Long System Prompt?**
A single massive system prompt becomes fragile, hard to test, and impossible to reuse. Skills decompose agent behavior into focused, testable units. You can swap, update, or combine skills without rewriting everything.

**Skill Anatomy**
Every skill has four sections:
1. **Role** — one sentence defining what the agent is in this context
2. **Trigger** — when this skill activates
3. **Rules** — explicit behavior rules (numbered list)
4. **API Calls** — which backend endpoints to call and when

## Deep Dive

**Writing Your First Skill**
```markdown
# concept-explainer

## Role
You are a patient tutor who explains AI Agent Development concepts using analogies and examples.

## Trigger
Activate when the student asks "what is", "explain", "how does", or "I don't understand".

## Rules
1. ALWAYS call GET /api/v1/search?q={concept} before explaining
2. Ground every explanation in the returned chunks
3. Cite the chapter number: "As covered in Chapter N..."
4. If search returns no results, say: "This topic is not in the current course material."
5. Adjust depth: ask "Beginner, Intermediate, or Advanced?" if unclear

## API Calls
- Search: GET /api/v1/search?q={student_question}&limit=3
- Chapter: GET /api/v1/chapters/{chapter_id} (for full context)
```

**Loading Skills into a ChatGPT App**
Skills are referenced in your `manifest.yaml` and injected into the system prompt at runtime:
```yaml
skills:
  - name: concept-explainer
    path: SKILLS/concept-explainer.md
    trigger: explanation_request
  - name: quiz-master
    path: SKILLS/quiz-master.md
    trigger: quiz_request
```

**Skill Composition**
For complex workflows, activate multiple skills in sequence. For example, the `socratic-tutor` skill triggers the `concept-explainer` skill after 3 failed student attempts:
```markdown
## Escalation Rule
After 3 exchanges without resolution, switch to concept-explainer mode.
```

## Code Example

```python
# Loading a skill file in Python (for testing or custom agents)
from pathlib import Path

def load_skill(skill_name: str) -> str:
    """Load a skill file and return its content as a string."""
    skill_path = Path("SKILLS") / f"{skill_name}.md"
    if not skill_path.exists():
        raise FileNotFoundError(f"Skill not found: {skill_name}")
    return skill_path.read_text(encoding="utf-8")

def build_system_prompt(base_prompt: str, skill_names: list[str]) -> str:
    """Combine base prompt with one or more skills."""
    skills_content = []
    for name in skill_names:
        skill = load_skill(name)
        skills_content.append(f"## Active Skill: {name}\n{skill}")

    return base_prompt + "\n\n" + "\n\n".join(skills_content)

# Usage
system = build_system_prompt(
    "You are a Course Companion for AI Agent Development.",
    ["concept-explainer", "progress-motivator"]
)
print(system[:500])
```

## Summary

- Skills are markdown files that modularize agent behavior
- Each skill has Role, Trigger, Rules, and API Calls sections
- Skills replace monolithic system prompts with focused, testable units
- Skills are loaded at runtime via the ChatGPT App manifest
- Multiple skills can be composed for complex agent workflows
- This project uses 4 skills: concept-explainer, quiz-master, socratic-tutor, progress-motivator
