# Claude Agent SDK Basics

## Overview

The Claude Agent SDK is Anthropic's Python library for building agents powered by Claude models. It gives you a clean interface for sending messages, defining tools, managing conversation history, and receiving structured responses. This chapter covers everything you need to write your first working agent — from installation to your first multi-turn conversation.

## Key Concepts

**Installation**
The Claude Agent SDK is published under the `anthropic` package on PyPI. Install it with:
```bash
pip install anthropic
```

**Message Roles**
Every conversation in Claude SDK is a list of messages. Each message has a role:
- `user` — messages from the human (or from tool results)
- `assistant` — messages from Claude
- `system` — the instruction that defines Claude's identity and behavior (passed separately, not in the messages list)

**System Prompt**
The system prompt is the most powerful part of your agent setup. It defines:
- Who the agent is ("You are a research assistant...")
- What it can and cannot do
- Its communication style and output format
- Which tools it should use and when

## Deep Dive

**Your First API Call**
```python
import anthropic

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from environment

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system="You are a helpful tutor for AI Agent Development.",
    messages=[
        {"role": "user", "content": "What is an AI agent?"}
    ]
)

print(response.content[0].text)
```

**Multi-Turn Conversation**
To maintain a conversation, append each response to the messages list and send the full history on every call:
```python
messages = []

def chat(user_input: str) -> str:
    messages.append({"role": "user", "content": user_input})
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system="You are a helpful tutor.",
        messages=messages,
    )
    reply = response.content[0].text
    messages.append({"role": "assistant", "content": reply})
    return reply
```

**Understanding Tokens**
Claude charges per token (roughly 4 characters = 1 token). The `max_tokens` parameter sets the ceiling on the response length. Monitor `response.usage` to track input and output tokens for cost control.

## Code Example

```python
import anthropic
import os

client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

messages = [{"role": "user", "content": "Explain what MCP is in one sentence."}]

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=200,
    system="You are an expert on AI Agent Development. Be concise.",
    messages=messages,
)

print(response.content[0].text)
print(f"Tokens used — input: {response.usage.input_tokens}, output: {response.usage.output_tokens}")
```

## Summary

- Install with `pip install anthropic`
- Three message roles: `system`, `user`, `assistant`
- System prompt defines agent identity and behavior
- Multi-turn conversations require sending full message history each call
- Monitor `response.usage` for token costs
- `max_tokens` controls maximum response length
