# Multi-Agent Patterns

## Overview

A single agent has limits: context window size, task specialization, and parallel execution. Multi-agent systems overcome these limits by distributing work across multiple specialized agents. This chapter covers the three core multi-agent patterns — Orchestrator-Worker, Pipeline, and Parallel Fan-Out — and shows you when to use each one.

## Key Concepts

**Why Multiple Agents?**
- **Context limits**: Break a 100,000-token task into chunks each agent can handle
- **Specialization**: A research agent, a writing agent, and a review agent each excel at one thing
- **Parallelism**: Run 10 sub-tasks simultaneously instead of sequentially
- **Isolation**: One agent's failure doesn't corrupt another agent's state

**Three Core Patterns**

| Pattern | Description | Best For |
|---------|-------------|----------|
| Orchestrator-Worker | One general agent delegates to specialist agents | Complex tasks with known sub-tasks |
| Pipeline | Output of Agent A becomes input to Agent B | Sequential transformations |
| Parallel Fan-Out | Same input sent to N agents simultaneously | Independent sub-tasks |

**Agent Communication**
Agents communicate by passing text. The orchestrator formats instructions for sub-agents; sub-agents return structured results. There is no shared memory — all state passes through messages.

## Deep Dive

**Pattern 1: Orchestrator-Worker**
```python
import anthropic

client = anthropic.Anthropic()

def research_agent(topic: str) -> str:
    """Specialized agent: finds facts about a topic."""
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=512,
        system="You are a research agent. Find and summarize key facts. Be concise.",
        messages=[{"role": "user", "content": f"Research: {topic}"}],
    )
    return response.content[0].text

def writing_agent(facts: str, format: str) -> str:
    """Specialized agent: turns facts into polished prose."""
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system=f"You are a writing agent. Write in {format} format. Use only the provided facts.",
        messages=[{"role": "user", "content": f"Facts:\n{facts}"}],
    )
    return response.content[0].text

def orchestrator(user_goal: str) -> str:
    """General agent that coordinates research and writing."""
    # Step 1: Research
    facts = research_agent(user_goal)
    # Step 2: Write
    article = writing_agent(facts, format="blog post")
    return article
```

**Pattern 2: Pipeline**
```python
def review_agent(draft: str) -> str:
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=256,
        system="You are a review agent. Output only: APPROVED or NEEDS_REVISION: <reason>",
        messages=[{"role": "user", "content": draft}],
    )
    return response.content[0].text

def pipeline(topic: str) -> str:
    """Research → Write → Review pipeline."""
    facts = research_agent(topic)
    draft = writing_agent(facts, "technical report")
    review = review_agent(draft)

    if review.startswith("APPROVED"):
        return draft
    else:
        # Revision loop
        reason = review.replace("NEEDS_REVISION:", "").strip()
        return writing_agent(f"{facts}\n\nRevision needed: {reason}", "technical report")
```

**Pattern 3: Parallel Fan-Out**
```python
import asyncio

async def parallel_research(topics: list[str]) -> list[str]:
    """Research multiple topics in parallel."""
    async def research_one(topic: str) -> str:
        # In production: use async anthropic client
        return research_agent(topic)

    tasks = [research_one(topic) for topic in topics]
    return await asyncio.gather(*tasks)
```

## Code Example

```python
# Agent Factory pattern: orchestrator creates custom agents from specs
def agent_factory(spec: str) -> callable:
    """Create a specialized agent from a spec string."""
    def custom_agent(task: str) -> str:
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            system=f"You are a specialized agent.\n\nSpec:\n{spec}",
            messages=[{"role": "user", "content": task}],
        )
        return response.content[0].text
    return custom_agent

# Create agents from specs
tutor_spec = open("SKILLS/concept-explainer.md").read()
tutor_agent = agent_factory(tutor_spec)
print(tutor_agent("Explain the MCP protocol"))
```

## Summary

- Multi-agent systems overcome context, specialization, and parallelism limits
- Three patterns: Orchestrator-Worker, Pipeline, Parallel Fan-Out
- Agents communicate via text messages — no shared memory
- Orchestrator pattern: one general agent delegates to specialists
- Pipeline pattern: output of one agent feeds the next
- Fan-out pattern: same task sent to multiple agents simultaneously
- Agent Factory: dynamically create specialized agents from spec files
