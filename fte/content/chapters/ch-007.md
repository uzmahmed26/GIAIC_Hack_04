# Agent Memory Systems

## Overview

By default, Claude has no memory between API calls. Every conversation starts fresh. For a Digital FTE to work effectively — remembering student progress, past conversations, and learned preferences — you need to architect explicit memory systems. This chapter covers the four types of agent memory and shows you how to implement each one.

## Key Concepts

**Four Types of Agent Memory**

| Type | What It Stores | Where It Lives | Lifespan |
|------|---------------|----------------|----------|
| In-Context | Current conversation | Messages array | Single session |
| External | Facts, history, documents | PostgreSQL / files | Persistent |
| Semantic | Meaning and concepts | pgvector embeddings | Persistent |
| Episodic | Past interactions | Structured DB records | Persistent |

**When to Use Each**
- **In-context**: Current task state, last few messages
- **External**: User profile, preferences, completed chapters
- **Semantic**: "What did we discuss about MCP?" — similarity search
- **Episodic**: "Last Tuesday you got 80% on Quiz 3" — exact recall

**The Memory-Augmented Agent**
A memory-augmented agent follows this pattern:
1. Query memory before generating a response
2. Inject relevant memories into the system prompt or context
3. Update memory after each significant interaction

## Deep Dive

**In-Context Memory (Basic)**
```python
# Already covered in Chapter 2 — the messages array IS in-context memory
messages = []

def chat_with_memory(user_input: str) -> str:
    messages.append({"role": "user", "content": user_input})
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system="You are a course tutor.",
        messages=messages[-20:],  # Keep last 20 messages to avoid context overflow
    )
    reply = response.content[0].text
    messages.append({"role": "assistant", "content": reply})
    return reply
```

**External Memory (PostgreSQL)**
```python
import asyncpg

async def save_memory(db, user_id: str, key: str, value: str):
    """Save a key-value memory for a user."""
    await db.execute(
        """
        INSERT INTO user_memories (user_id, key, value, created_at)
        VALUES ($1, $2, $3, NOW())
        ON CONFLICT (user_id, key) DO UPDATE SET value = EXCLUDED.value
        """,
        user_id, key, value
    )

async def load_memories(db, user_id: str) -> dict:
    """Load all memories for a user."""
    rows = await db.fetch(
        "SELECT key, value FROM user_memories WHERE user_id = $1", user_id
    )
    return {row["key"]: row["value"] for row in rows}

# Usage: inject into system prompt
memories = await load_memories(db, "user123")
memory_context = "\n".join(f"- {k}: {v}" for k, v in memories.items())
system_prompt = f"You are a tutor.\n\nStudent memories:\n{memory_context}"
```

**Semantic Memory (pgvector)**
```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")

async def remember_interaction(db, user_id: str, content: str):
    """Store an interaction with its embedding for semantic search."""
    embedding = model.encode(content).tolist()
    await db.execute(
        """
        INSERT INTO interaction_memories (user_id, content, embedding, created_at)
        VALUES ($1, $2, $3::vector, NOW())
        """,
        user_id, content, embedding
    )

async def recall_similar(db, user_id: str, query: str, limit: int = 3) -> list:
    """Recall past interactions semantically similar to the query."""
    query_embedding = model.encode(query).tolist()
    rows = await db.fetch(
        """
        SELECT content, 1 - (embedding <=> $1::vector) AS similarity
        FROM interaction_memories
        WHERE user_id = $2
        ORDER BY embedding <=> $1::vector
        LIMIT $3
        """,
        query_embedding, user_id, limit
    )
    return [{"content": r["content"], "similarity": r["similarity"]} for r in rows]
```

## Code Example

```python
# Memory-augmented tutor agent
import anthropic

client = anthropic.Anthropic()

async def memory_augmented_tutor(db, user_id: str, question: str) -> str:
    """Answer a question using relevant past interactions as context."""

    # 1. Recall semantically similar past interactions
    memories = await recall_similar(db, user_id, question, limit=3)
    memory_text = "\n".join(
        f"[Past: {m['content'][:100]}...]" for m in memories
    ) if memories else "No relevant past interactions."

    # 2. Load progress as external memory
    row = await db.fetchrow(
        "SELECT chapters_completed FROM user_summary WHERE user_id = $1", user_id
    )
    progress = f"Completed chapters: {row['chapters_completed']}" if row else "New student"

    # 3. Build enriched system prompt
    system = f"""You are a patient course tutor.

Student context:
- {progress}
- Relevant past interactions:
{memory_text}

Use this context to personalize your answer."""

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=512,
        system=system,
        messages=[{"role": "user", "content": question}],
    )

    # 4. Save this interaction to memory
    await remember_interaction(db, user_id, f"Q: {question}")

    return response.content[0].text
```

## Summary

- Four memory types: In-Context, External, Semantic, Episodic
- In-context memory = the messages array (single session only)
- External memory = PostgreSQL key-value store (persistent facts)
- Semantic memory = pgvector embeddings (similarity-based recall)
- Inject memories into system prompt to personalize responses
- Always save new interactions back to memory after responding
- Limit in-context messages to avoid exceeding the context window
